# Contributing Guide

Thank you for contributing to `bash-ask`.

## How to Contribute

- Fork the repository and create a new branch for your change.
- Keep changes focused and small when possible.
- Test the script manually before submitting a pull request.
- Update documentation if behavior or setup changes.
- Use clear commit messages that explain the purpose of the change.

## Code Style

- Keep the script compatible with Bash.
- Prefer simple, readable shell code.
- Reuse the existing style and indentation in `ask`.
- Avoid adding unnecessary complexity or extra dependencies.

## Before Opening a Pull Request

- Check that the script still handles arguments and piped input correctly.
- Make sure required environment variables are still validated clearly.
- Confirm API requests and output handling still behave as expected.
- Document any limitation, assumption, or new behavior in `README.md`.
